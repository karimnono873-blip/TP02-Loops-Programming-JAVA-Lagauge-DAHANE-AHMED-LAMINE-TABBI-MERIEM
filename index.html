1. Objectif du TP

Le but fondamental de ce deuxième travail pratique est d'apprendre, de comprendre et de maîtriser la programmation des structures itératives (communément appelées "boucles") par l'utilisation du langage Java.

Dans la Programmation Orientée Objet (POO) et dans le développement logiciel en général, les boucles sont des outils indispensables. Elles permettent d'exécuter un bloc d'instructions de manière répétée et contrôlée selon une ou plusieurs conditions définies par le programmeur. L'utilisation correcte des boucles permet de :

  • Éviter la redondance : Ne pas écrire 100 fois la même ligne de code.
  • Automatiser des tâches : Traiter de grandes quantités de données (comme des tableaux ou des listes).
  • Créer des interactions continues : Maintenir un programme ouvert en attendant l'action d'un utilisateur, ce qui est particulièrement utile dans les systèmes embarqués ou les interfaces graphiques en Automatique.

2.1 La boucle for en Java

La boucle for est sans doute la boucle la plus utilisée lorsqu'il s'agit de parcourir des éléments dont le nombre est connu à l'avance. Sa syntaxe compacte regroupe toute la logique de comptage sur une seule ligne.

La structure générale de la boucle for est :

for (<initialisation>; <condition>; <pas>) { <instructions> }
Exemple_Boucle_For.java Java 
public class ExempleFor {
    public static void main(String[] args) {
        // i commence à 1 ; la boucle tourne tant que i <= 5 ; i augmente de +1 à chaque tour
        for (int i = 1; i <= 5; i++) {
            System.out.print(i + " ");
        }
        // Retour à la ligne après l'exécution de la boucle
        System.out.println();
    }
}

Explication détaillée de l'exécution :

Mécanisme interne : pourquoi affiche-t-il exactement "1 2 3 4 5 " ?

  1. Initialisation : La machine lit int i = 1 une seule fois au tout début. La mémoire alloue un espace pour la variable i et lui donne la valeur 1.
  2. Évaluation : Elle vérifie la condition i <= 5. Puisque 1 est inférieur ou égal à 5, la condition est vraie (true).
  3. Exécution : Elle exécute le corps de la boucle, imprimant "1 ".
  4. Incrémentation (Le pas) : Elle remonte à la signature de la boucle et exécute i++. La variable i passe à 2.
  5. Le cycle (Évaluation -> Exécution -> Incrémentation) se répète jusqu'à ce que i devienne 6.
  6. Quand i = 6, la condition 6 <= 5 est fausse (false). La boucle s'arrête instantanément sans exécuter le code intérieur.

Avantage majeur : La gestion du compteur est 100% automatique. On utilise cette boucle lorsqu'on doit répéter une action un nombre de fois prédéfini et connu à l'avance.

2.2 La boucle while (Tant Que)

La boucle while est la structure itérative la plus basique. Contrairement à la boucle for, elle ne gère pas de compteur intégré. Elle se contente d'évaluer une condition booléenne.

La structure générale de cette boucle est extrêmement simple :

while (<condition>) { <instructions> }
Exemple_Boucle_While.java Java 
public class ExempleWhile {
    public static void main(String[] args) {
        // 1. Initialisation manuelle du compteur AVANT la boucle
        int i = 1; 
        
        // 2. Contrôle de la condition de poursuite
        while (i <= 5) {
            System.out.print(i + " ");
            
            // 3. Incrémentation manuelle DANS le corps de la boucle
            i++; 
        }
        System.out.println();
    }
}

Explication détaillée et Pièges à éviter :

Différence fondamentale avec le FOR :
Bien que le résultat affiché ("1 2 3 4 5 ") soit strictement identique à l'exemple précédent, l'architecture du code est très différente. La boucle while nécessite une discipline stricte de la part du développeur.

N'oubliez jamais lorsqu'une boucle while fonctionne avec un compteur, il faut respecter ces trois règles d'or de l'algorithmique :

  • Règle 1 : Il faut obligatoirement initialiser le compteur (ici int i = 1;) avant d'entrer dans la boucle.
  • Règle 2 : Il faut contrôler correctement la valeur du compteur dans la condition de boucle (ici i <= 5).
  • Règle 3 (CRITIQUE) : Il faut absolument incrémenter le compteur à la fin du corps de la boucle (le i++;). Si vous oubliez cette ligne, i vaudra toujours 1. La condition 1 <= 5 sera toujours vraie à l'infini. C'est ce qu'on appelle une Boucle Infinie, qui fait planter ou geler le programme (dépassement de capacité mémoire / CPU).

Note syntaxique de Java : Notez bien qu'il n'y a pas de point-virgule (;) après la parenthèse fermante du while(...).

2.3 La boucle do...while (Répéter... Jusqu'à)

La boucle do...while est une variante très spécifique de la boucle while. Elle inverse le processus de vérification.

La structure de la boucle do...while est :

do { <instructions> } while(<condition>);
Exemple_Boucle_Do_While.java Java 
public class ExempleDoWhile {
    public static void main(String[] args) {
        int i = 1;
        
        do {
            // L'incrémentation se fait AVANT l'affichage dans cet exemple précis
            i++; 
            System.out.print(i + " ");
            
        // La condition est évaluée à la FIN, notez la présence du point-virgule obligatoire ici
        } while (i <= 5); 
        
        System.out.println();
    }
}

Analyse d'exécution et Logique Inverse :

Pourquoi affiche-t-il "2 3 4 5 6 " et non pas "1 2 3 4 5" ?

C'est ici qu'il faut être très attentif à l'ordre des instructions dans le code écrit dans l'énoncé du TP :

  • La variable i est initialisée à 1.
  • La machine entre dans le bloc do. Elle lit l'instruction i++. Donc i devient 2.
  • Ensuite, elle exécute System.out.print(i). Le premier chiffre affiché est donc bien 2.
  • La boucle continue. Lorsque i vaut 5, on affiche 5. Ensuite, l'évaluation while (5 <= 5) est vraie. La boucle recommence donc une dernière fois !
  • Le i passe à 6 grâce au i++, la machine affiche 6, et enfin l'évaluation while (6 <= 5) devient fausse. La boucle s'arrête.

La règle d'or du Do...While :
La condition est évaluée après chaque passage dans la boucle. La conséquence directe et fondamentale en programmation est que l'on exécute toujours au moins une fois les instructions contenues dans une boucle do...while (Répéter... jusqu'à), même si la condition de fin est fausse dès le départ.

3. Exercice : Conception d'un Système d'Authentification

L'Énoncé détaillé :

Écrire un programme complet en langage JAVA qui permet de vérifier le mot de passe saisi au clavier par un utilisateur. La politique de sécurité est la suivante :
  • L'utilisateur a droit à un maximum de 3 chances. S'il réussit, la machine lui affiche le succès de l'authentification.
  • Si les 3 chances sont épuisées, un message strict de "Compte bloqué" sera affiché à l'écran.
  • Mécanisme de récupération : Dans le cas où l'utilisateur a bloqué son compte, le système lui propose une ultime chance de retenter l'accès sous forme de "question secrète" à laquelle il devra répondre (O/N pour accepter, puis saisie de la réponse).
  • Si la réponse à la question secrète est incorrecte, l'accès lui est définitivement refusé, le programme s'arrête.

Remarque technique : Le mot de passe correct attendu dans la base de données est 'AIIMI' et la réponse correcte à la question secrète est 'Auto'.

Code Source Java (Implémentation de l'algorithme)

AuthentificationSysteme.java Java 
import java.util.Scanner; // Nécessaire pour capturer les frappes clavier

public class AuthentificationSysteme {
    
    public static void main(String[] args) {
        
        // 1. Initialisation du flux de lecture
        Scanner clavier = new Scanner(System.in);
        
        // 2. Définition des données de sécurité "en dur" (depuis l'énoncé)
        final String MOT_DE_PASSE_VALIDE = "AIIMI";
        final String REPONSE_SECRETE_VALIDE = "Auto";
        
        // 3. Variables d'état (La machine à états)
        int compteurErreurs = 0;
        boolean isAuthentifie = false;
        String saisie = "";
        
        /* =========================================================
           PHASE 1 : La Boucle de saisie du mot de passe
           On boucle TANT QUE les erreurs sont < 3 ET que l'utilisateur 
           n'est pas encore authentifié.
           ========================================================= */
        while (compteurErreurs < 3 && !isAuthentifie) {
            
            System.out.print("Veuillez saisir le mot de passe : ");
            saisie = clavier.nextLine();
            
            // Comparaison Orientée Objet : On utilise la méthode equals()
            if (saisie.equals(MOT_DE_PASSE_VALIDE)) {
                
                // Le booléen passe à true, ce qui va FORCER la sortie du while
                isAuthentifie = true; 
                
            } else {
                
                compteurErreurs++; // On incrémente le nombre d'échecs
                
                // On n'affiche le message d'erreur que s'il reste des tentatives
                if (compteurErreurs < 3) {
                    int chancesRestantes = 3 - compteurErreurs;
                    System.out.println("Mot de passe incorrect. Il vous reste " 
                                       + chancesRestantes + " chance(s).");
                }
            }
        }
        
        /* =========================================================
           PHASE 2 : Analyse Post-Boucle (Que s'est-il passé ?)
           On sort de la boucle soit par succès, soit par blocage.
           ========================================================= */
        if (isAuthentifie) {
            
            // L'utilisateur a réussi (du 1er, 2e ou 3e coup)
            System.out.println("\n[+] Succès de l'authentification. Accès Autorisé.");
            
        } else {
            
            // La boucle a cassé car compteurErreurs == 3
            System.out.println("\n[!] COMPTE BLOQUÉ. Nombre de tentatives maximum atteint.");
            
            /* =====================================================
               PHASE 3 : Procédure de récupération (Question secrète)
               ===================================================== */
            System.out.print("Voulez-vous utiliser la question secrète pour débloquer ? (O/N) : ");
            String choixRecuperation = clavier.nextLine();
            
            // equalsIgnoreCase() permet d'accepter 'o' ou 'O' indifféremment
            if (choixRecuperation.equalsIgnoreCase("O")) {
                
                System.out.print("Question Secrète : Quelle est la spécialité étudiée ? ");
                String reponseFournie = clavier.nextLine();
                
                if (reponseFournie.equals(REPONSE_SECRETE_VALIDE)) {
                    System.out.println("[+] Succès ! Compte débloqué via la question secrète.");
                } else {
                    System.out.println("[-] Réponse incorrecte. Accès définitivement refusé.");
                }
                
            } else {
                // Si l'utilisateur tape 'N' ou n'importe quelle autre touche
                System.out.println("[-] Procédure de secours annulée. Accès définitivement refusé.");
            }
        }
        
        // Fermeture du flux pour éviter les fuites de mémoire (Memory Leaks)
        clavier.close();
    }
}

Explication approfondie des choix architecturaux :

Pourquoi ce code est-il optimisé et professionnel ? Décortiquons la logique :

1. La condition combinée du while :
La ligne magique de cet algorithme est while (compteurErreurs < 3 && !isAuthentifie). L'opérateur logique && (ET) agit comme un filtre puissant. Si l'utilisateur tape le bon mot de passe ("AIIMI") au premier essai, le booléen isAuthentifie passe à true. De ce fait, l'expression !isAuthentifie (NON authentifié) devient false. La boucle est immédiatement brisée, évitant de demander inutilement un mot de passe déjà validé. C'est l'essence même de l'optimisation algorithmique.

2. La Comparaison des Objets String en POO :
C'est une erreur classique des débutants en Java de vouloir utiliser l'opérateur d'égalité == pour comparer du texte (ex: saisie == "AIIMI"). En Orienté Objet, la variable saisie est une référence vers un espace mémoire. L'opérateur == comparerait donc les adresses mémoires (qui sont différentes), et non pas les lettres tapées ! C'est pourquoi nous sommes obligés d'appeler la méthode .equals() de la classe String, qui va parcourir le texte caractère par caractère pour assurer la correspondance.

3. Robustesse avec equalsIgnoreCase() :
Dans la phase de récupération, l'utilisateur doit répondre (O/N). Il pourrait taper un 'o' minuscule. Au lieu d'écrire une condition lourde comme if (choix.equals("O") || choix.equals("o")), l'API Java nous offre equalsIgnoreCase(). Cette méthode ignore la casse (majuscule/minuscule), rendant notre programme beaucoup plus robuste face aux erreurs de frappe de l'utilisateur final.
