1. Objectif du TP

Le but principal de ce deuxième travail pratique est d'apprendre et de maîtriser la programmation des structures itératives (boucles) par l'utilisation du langage Java.

En Programmation Orientée Objet (POO), les boucles sont essentielles. Elles permettent d'exécuter un bloc d'instructions de manière répétée selon une condition donnée. Cela évite la redondance du code, optimise l'utilisation de la mémoire, et permet de traiter des flux de données (comme la lecture de capteurs en Automatique) ou de gérer des interactions utilisateur continues (comme un système d'authentification).

2. Programmation des boucles en Java

L'énoncé du TP souligne qu'il existe trois types principaux de boucles en Java : while, do... while, et for. Le choix de la boucle adéquate dépend fortement de la logique de l'algorithme à implémenter.

2.1 La boucle for en Java

La structure générale de la boucle for est définie comme suit :

for (<initialisation>; <condition>; <pas>) {
    <instructions>
}
💡 Comment fonctionne-t-elle et quels sont ses avantages ?
  • La gestion du compteur est automatique et centralisée sur une seule ligne (initialisation, test de la condition, et incrémentation).
  • Avantage majeur : Il faut utiliser cette boucle lorsqu'on connaît à l'avance le nombre exact d'itérations à effectuer (par exemple, parcourir un tableau de taille fixe).

Exemple de l'énoncé :

public static void main(String[] args) {
    for (int i = 1; i <= 5; i++) {
        System.out.print(i + " ");
    }
    System.out.println();
}

2.2 La boucle while (Tant Que)

La structure générale de cette boucle est très simple :

while (<condition>) {
    <instructions>
}
⚠️ Règles critiques de la boucle while :
Contrairement à la boucle for, la boucle while ne gère pas son propre compteur. Il faut donc impérativement respecter ces étapes :
  1. Initialiser le compteur avant d'entrer dans la boucle.
  2. Contrôler la valeur du compteur dans la condition de boucle.
  3. Incrémenter (ou modifier) le compteur à la fin du corps de la boucle, sinon le programme entrera dans une boucle infinie.
Notez bien qu'il n'y a pas de point-virgule après l'instruction while.

2.3 La boucle do...while (Répéter... Jusqu'à)

La structure de la boucle do...while (répéter) est :

do {
    <instructions>
} while (<condition>);
✅ L'avantage unique du do...while :
La condition est évaluée après chaque passage dans la boucle. On exécute le corps de la boucle tant que la condition est vérifiée. L'énorme différence avec le while normal est que l'on passe toujours au moins une fois dans une boucle do...while, peu importe si la condition est vraie ou fausse à l'origine. C'est idéal pour demander une saisie à un utilisateur (menus interactifs, demande de mot de passe).

3. Travail Demandé : Système d'Authentification Sécurisé

Énoncé de l'exercice : Écrire un programme en JAVA qui permet de vérifier le mot de passe saisi au clavier. L'utilisateur a droit à 3 chances pour que la machine lui affiche le succès de l'authentification sinon un message de compte bloqué sera affiché. Dans le cas où il veut encore retenter l'accès au compte une nouvelle chance lui est proposée sous forme de question secrète à laquelle il devra répondre. Si la réponse est incorrecte l'accès lui est définitivement refusé pour l'exécution en cours.

Rappel des consignes : le mot de passe correct est 'AIIMI' et la réponse correcte à la question secrète est 'Auto'.

3.1 Analyse Algorithmique et Stratégie (Comment concevoir la solution ?)

Pour résoudre ce problème de manière professionnelle en POO, nous devons décomposer la logique (Machine à états) :

  • État initial : L'utilisateur n'est pas connecté. Il a 0 tentative à son actif.
  • Le choix de la boucle : L'utilisateur doit saisir un mot de passe et on doit vérifier s'il a bon. S'il a faux, il recommence, jusqu'à un maximum de 3 fois. Nous allons utiliser une boucle while avec une condition complexe (composée) : on continue la boucle TANT QUE le nombre de tentatives est inférieur à 3 ET que l'utilisateur n'est pas authentifié.
  • Le Mécanisme de blocage : À la sortie de la boucle, il y a deux possibilités.
    1) La boucle s'est arrêtée car le mot de passe est bon (Succès).
    2) La boucle s'est arrêtée car le compteur a atteint 3 (Échec).
  • Le Mécanisme de secours (Question secrète) : Si on est dans le cas de l'échec (2), on affiche le blocage, puis on utilise un simple if/else pour proposer la question secrète. Si la réponse est "Auto", on valide.

3.2 Implémentation du Code Source Java

Authentification.java Java 
import java.util.Scanner; // Importation pour lire les entrées clavier

public class Authentification {
    public static void main(String[] args) {
        Scanner lecteur = new Scanner(System.in);
        
        // 1. Déclaration des variables et constantes (définies dans l'énoncé)
        String mdpCorrect = "AIIMI";
        String reponseSecreteCorrecte = "Auto";
        
        int nombreTentatives = 0;
        boolean estAuthentifie = false;
        String saisieUtilisateur = "";
        
        // 2. Boucle de vérification principale (Max 3 chances)
        // La boucle tourne tant qu'on a moins de 3 tentatives ET qu'on n'est pas authentifié
        while (nombreTentatives < 3 && !estAuthentifie) {
            System.out.print("Veuillez saisir le mot de passe : ");
            saisieUtilisateur = lecteur.nextLine();
            
            // Utilisation de la méthode equals() pour comparer le contenu des String
            if (saisieUtilisateur.equals(mdpCorrect)) {
                estAuthentifie = true; // Le booléen passe à true, ce qui va stopper la boucle
            } else {
                nombreTentatives++; // Incrémentation du compteur d'échec
                
                // Affichage du nombre de chances restantes si ce n'est pas le dernier essai
                if (nombreTentatives < 3) {
                    System.out.println("Erreur. Il vous reste " + (3 - nombreTentatives) + " tentative(s).");
                }
            }
        }
        
        // 3. Traitement POST-BOUCLE : Vérification de l'état final
        if (estAuthentifie) {
            // Cas 1 : L'utilisateur a trouvé le mot de passe dans les 3 essais
            System.out.println(">> Succès de l'authentification. Bienvenue dans le système !");
        } else {
            // Cas 2 : L'utilisateur a épuisé ses 3 chances (nombreTentatives == 3)
            System.out.println(">> COMPTE BLOQUÉ. Nombre de tentatives maximum atteint.");
            
            // Mise en place de l'alternative de secours
            System.out.print("Voulez-vous répondre à la question secrète pour débloquer ? (O/N) : ");
            String choix = lecteur.nextLine();
            
            // equalsIgnoreCase() permet d'accepter 'o' minuscule ou 'O' majuscule
            if (choix.equalsIgnoreCase("O")) {
                System.out.print("Question Secrète : Quelle est la spécialité étudiée ? ");
                String repSecrete = lecteur.nextLine();
                
                if (repSecrete.equals(reponseSecreteCorrecte)) {
                    System.out.println(">> Succès ! Compte débloqué via la question secrète.");
                } else {
                    System.out.println(">> Réponse incorrecte. Accès définitivement refusé.");
                }
            } else {
                System.out.println(">> Procédure annulée. Accès définitivement refusé.");
            }
        }
        
        // Libération de la mémoire du Scanner
        lecteur.close();
    }
}

3.3 Explications détaillées (Les touches techniques du programme)

Pourquoi utiliser equals() et non pas == pour les chaînes de caractères ?
C'est une spécificité cruciale en Java. Une variable de type String est un Objet. Si on utilise l'opérateur ==, Java va comparer les adresses mémoire des deux objets, et non pas le texte qu'ils contiennent. Pour vérifier si le texte tapé correspond à "AIIMI", nous appelons la méthode .equals() de la classe String qui compare le contenu lettre par lettre.

L'avantage de la condition composée dans le while :

La ligne while (nombreTentatives < 3 && !estAuthentifie) est le cœur de notre logique d'optimisation.

  • L'opérateur logique && (ET) exige que les deux conditions soient vraies pour que la boucle continue.
  • Si l'utilisateur tape le bon mot de passe au premier essai, estAuthentifie devient true. Le test !estAuthentifie (NON authentifié) devient false. La boucle s'arrête immédiatement. L'algorithme est performant car il ne pose pas la question inutilement 3 fois.

L'utilisation de equalsIgnoreCase() :

Lors de la demande (O/N), l'utilisateur pourrait taper un 'o' minuscule ou un 'O' majuscule. Au lieu de faire un test complexe if (choix.equals("O") || choix.equals("o")), l'API Java fournit la méthode equalsIgnoreCase() qui gère cette robustesse automatiquement. C'est un grand avantage en termes de lisibilité du code.
